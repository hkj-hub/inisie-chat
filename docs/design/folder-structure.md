# フォルダ構成

## 想定読者と目的

- **想定読者**: このリポジトリに初めて触れる開発者、および実装を担当する SubAgent
- **目的**: 各ディレクトリの責務を明確にし、ファイルをどこに置くべきかの判断基準を提供する
- **スコープ**: リポジトリのディレクトリ構成と各フォルダの責務。ファイル名・関数名等の実装詳細は含まない

---

## トップレベル構成

```txt
inisie-chat/
├── apps/
│   ├── backend/        # Node.js + Hono API（Port 3000）
│   └── frontend/       # Vue.js 3 SPA（Port 5173）
├── packages/
│   └── shared/         # フロントエンド・バックエンド共通の型・schema
├── data/               # CSV データファイル
├── docs/               # 設計書・ADR
└── .github/            # Copilot 設定・Agent・Skill・Prompt
```

### 各ディレクトリの責務

| ディレクトリ | 責務 |
|---|---|
| `apps/backend/` | Hono を使った HTTP API サーバー。チャット送信・取得・ログ取得を提供する |
| `apps/frontend/` | Vue.js 3 を使った SPA。エントランス・チャット・ログの 3 画面を持つ |
| `packages/shared/` | フロントエンドとバックエンドの両方が参照する型・zod スキーマ |
| `data/` | `chat.csv`（最新 20 件）と `log.csv`（全履歴）を格納する |
| `docs/` | ADR・API 仕様書・フォルダ構成ドキュメント等の設計記録 |
| `.github/` | GitHub Copilot・Agent・Skill・Prompt の設定ファイル群 |

### 配置判断の原則

- トップレベルは技術レイヤー名ではなく責務でディレクトリを切る
- `apps/` 以下は機能・ドメイン単位で内部構成を整理する
- `packages/shared/` に置くのは「フロントエンドとバックエンドの両方から使われる」ものだけとする
- ルート直下に `src/`・`lib/`・`components/` 等を作らない

## apps/backend の詳細構成

```txt
apps/backend/
├── src/
│   ├── chats/          # チャット送信・取得に関するすべて
│   ├── logs/           # ログ（全履歴）取得に関するすべて
│   └── shared/         # バックエンド内で複数機能から使われるユーティリティ
├── package.json
└── tsconfig.json
```

### 機能フォルダの構成原則

各機能フォルダ（`chats/`・`logs/`）は、以下を同一フォルダに近接配置する。

| 種別 | 役割 |
|---|---|
| ルートハンドラ定義 | リクエスト受け取り・レスポンス返却のみを担う薄いレイヤー |
| ユースケース関数 | ビジネスロジック（最新 20 件に制限する等）を担う |
| リポジトリ | CSV ファイルへのアクセスを隔離する |
| schema | zod による入力検証 schema |
| 型定義 | 機能固有の TypeScript 型 |
| テスト | ユースケース関数・リポジトリの単体テスト |

技術レイヤー別フォルダ（`handlers/`・`services/`・`repositories/` 等）を `src/` のトップレベルに作らない。これらの名前は各機能フォルダの内側でのみ使用する。

### chats/ の責務

- チャット一覧の取得（最新 20 件）
- チャストの送信（`chat.csv` へ追記 + 最新 20 件に刈り込み、`log.csv` へ追記）
- POST リクエストボディの入力検証

### logs/ の責務

- 全履歴チャットの取得（`log.csv` 全件）

### shared/ の責務

バックエンド内の複数機能から自然に使われるユーティリティを置く。
1 機能だけが使うコードは当該機能フォルダに置く。

- HTTP レスポンスのユーティリティ（成功・エラーフォーマットの組み立て）
- CSV ファイル読み書きの汎用クライアント
- 共通エラー型

## apps/frontend の詳細構成

```txt
apps/frontend/
├── public/
├── src/
│   ├── router/         # Vue Router 設定
│   ├── features/
│   │   ├── entrance/   # エントランス画面（名前入力・色選択・入室）
│   │   ├── chat/       # チャット画面（一覧・投稿・1 分ポーリング）
│   │   └── log/        # ログ画面（全履歴表示・読み取り専用）
│   ├── shared/         # フロントエンド内の機能横断ユーティリティ
│   ├── App.vue
│   └── main.ts
├── tests/
├── package.json
└── tsconfig.json
```

### features/ の構成原則

各機能フォルダ（`entrance/`・`chat/`・`log/`）は、以下を同一フォルダに近接配置する。

| 種別 | 役割 |
|---|---|
| ページコンポーネント | 画面レイアウトとデータ取得結果の表示のみを担う |
| データ取得関数 | API へのリクエストと zod によるレスポンス検証 |
| 機能固有の UI コンポーネント | その機能でのみ使われる Vue コンポーネント |

### entrance/ の責務

- 名前入力と色選択の受け付け
- 入室前のチャット閲覧（読み取り専用表示）
- チャット画面・ログ画面へのリンク提供

### chat/ の責務

- チャット一覧の表示
- コメント入力と送信
- 1 分に 1 回のポーリング更新（送信時は即時更新）

### log/ の責務

- 全チャット履歴の表示（読み取り専用）

### shared/ の責務

フロントエンド内の複数機能から自然に使われるものだけを置く。
「便利な置き場」として使わない。

- `shared/ui/`: サイト共通の UI コンポーネント（ヘッダー等）
- `shared/lib/`: 機能横断の純粋関数（日付フォーマット等）

機能専用のコンポーネントや関数は `shared/` ではなく各機能フォルダに置く。

## packages/shared の詳細構成

```txt
packages/shared/
├── src/
│   ├── types/          # TypeScript 型定義（ChatEntry / LogEntry）
│   └── schemas/        # zod スキーマ定義
└── package.json
```

### 配置基準

`packages/shared` に置くのは以下の条件を **すべて** 満たすものだけとする。

1. フロントエンドとバックエンドの **両方** から使われる
2. 特定のアプリや画面に依存しない
3. 型・schema・純粋関数として説明できる

Vue コンポーネント・Hono / Node.js に依存するコード・1 つのアプリだけで使うコードは置かない。

### types/ の責務

- `ChatEntry` 型: タイムスタンプ・名前・色・メッセージから成るチャット 1 件の型
- `LogEntry` 型: `ChatEntry` と同一構造のログ 1 件の型

### schemas/ の責務

- `chatEntrySchema`: `ChatEntry` の zod スキーマ（フィールドごとの制約を含む）
- `chatColorSchema`: 使用可能な 30 色の列挙 schema
- POST リクエストボディの検証 schema

## data/ の CSV フォーマット仕様

```txt
data/
├── chat.csv    # 最新 20 件のチャット
└── log.csv     # 全チャット履歴（無制限）
```

### CSV フォーマット

両ファイルとも同一のフォーマットを使用する。

```
timestamp,name,color,message
2026-05-19T12:34:56.000Z,Alice,blue,こんにちは！
2026-05-19T12:35:10.000Z,Bob,red,やあ！
```

| 列 | 型 | 制約 |
|---|---|---|
| `timestamp` | 文字列（ISO 8601） | UTC タイムゾーン |
| `name` | 文字列 | 1〜50 文字 |
| `color` | 文字列 | 30 色いずれか |
| `message` | 文字列 | 1〜500 文字 |

### ファイルの管理方針

- `chat.csv`: チャット投稿のたびに末尾に追記し、20 件を超えた場合は古い行を削除して最新 20 件のみ保持する
- `log.csv`: チャット投稿のたびに末尾に追記する。削除は行わない（全履歴を保持）
- 両ファイルのパスは環境変数で設定可能にし、テスト時には別ファイルを指定できる
